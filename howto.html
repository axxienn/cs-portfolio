HTML vs. CSS

  • Stands for the Hypertext Markup Language
  • It is the coding language for creating and structuring a website
  • It's also where you insert your visible informations and images onto the webpage
  • For example, a commonly used tag goes like this:
    • content
    • Most HTML elements contain an opening tag and a closing tag. The difference between the tags is that a closing tag has a slash after the opening angle bracket, < .
  • Stands for Cascading Style Sheets
  • CSS acts like an aid to HTML. It helps to construct the informations into something more organized and compelling
  • It's responsible for complex layouts and designs.
    • This includes the margin spaces, padding thickness, and floats
  • It's able to control and modify multiple pages at once. (CCS External Style Sheet)
    • saves time and work

HTML & CSS Basics

A basic HTML format for structuring a page looks like this:

  • Every HTML document must contain < html > and < /html >. It describes the HTML document.
  • The purpose of a < head > tag is to insert informations about the document.
  • The < title > tag is what you use to insert a title.
  • The < body > tag is used to contain the visible page contents.
  • The < h1 > means heading.
    • There is also < h2 >, < h3 >, all the way to < h6 >, rating from the least to most important.
  • The paragraph tag, < p >, indicates a paragraph.

A basic CSS format looks like this:

  • The h1 is known as the selector. It indicates that you are modifying specifically heading 1.
    • Then you would use an opening and closing braces, { }, to contain the modifications. The braces can also be identified as rulesets.
    • Everything inside of a ruleset are called declaration block(s). A ruleset can contain multiple declarations and every declaration block consists of a property and value.
  • To put the content in the center, insert text-align: center;
    • In coding, the text-align is known as a property. The function of a property is to specify your command. This includes changing the text alignment, font color, the space between one another, and so on.
      • After you've inserted a property, it must be followed by a colon. A colon lets the program know that it is the end of a property.
    • Every property follows by a value. The value is used to help the property to identify further that this is the certain change you want to make. In this case, the instruction- center -would be the value.
      • After every value, a semi-colon is necessary to conclude the declaration.
  • To change the font into bold, insert font-weight: bold;
  • To change the font-color into blue, insert color: blue;

How to code a menu

To create a menu, you need to first insert your contents. For the menu that I've created on my page, my HTML structure looks like this:

  • An < ul > tag stands for unordered list. Its function is to create or set off a bulleted list. There is also an ol tag, < ol >. It is an ordered list that set off a numbered list or procedures. In this case however, the ul tag is a better option than an ol tag because you do not need your menu to be numbered.
  • Labeling the ID as "navbar" will help CSS to identify that this is the area you want to modify.
  • A li tag, < li >, stands for list item. Its function is to insert the contents of a bullet point. Using both the unordered list tag and the list item tag will give you an unordered list. The li tag also works with ordered lists.
  • An < a > tag indicates a hyperlink. It allows you to link your current page to another page. The < a > tag is an important tag for creating a menu because bringing visitors to another page as a shortcut form is the whole purpose of a menu.
  • The href attribute is used to insert a page URL. The < a > tag needs a href attribute in order to be activated as a hyperlink.

Now that I've entered the contents of my menu, I would need to personalize it. As mentioned in CSS introduction, CSS allows me to modify the layouts and designs. The CCS structure of my menu looks like this:

  • Since the navigation bar was labeled as an ID in the HTML, you use a hashtag (#) in CSS to indicate that this is the section you want to modify.
  • To modify the list items more specifically, you would add li after #navbar. This way, every customization you make would direct only to the list items.
  • The display of inline-table is important if you want your tabs to be aligned. If this input is missing, the menu of your website would be in a listed form down the page.
  • The margin and padding functions are responsible for the spacing in between contents. Specifically, the margin's thickness determines the area outside of the border whereas the padding's thickness determines the area between the border and the contents.
  • In fact, the declaration blocks are relatively straightforward and self explanatory. For instance, the font size changes the word size while the background color changes the background color.
